# Value Functions

These are the repository-level value functions used to judge code, tests, docs, PRs, and model behavior.

## VF-1 Evidence traceability
Every output must point to input claims, evidence events, or provenance.

## VF-2 Falsifiability
Every elevated hypothesis must include alternative explanations and questions that could reduce confidence.

## VF-3 Calibration
Confidence must fall when evidence is weak, missing, noisy, or single-modality.

## VF-4 Conservative actionability
The system may recommend review questions, not punishment.

## VF-5 Reproducibility
Every demo run must be reproducible by one command.

## VF-6 Modularity
Heavy media tools are adapters; the core must remain deterministic and testable.

## VF-7 Human-review readiness
Reports must be readable by an engineer, investigator, journalist, or researcher without hidden model worship.

## VF-8 PR discipline
No PR merges unless tests, schemas, docs, and invariant gates pass.
